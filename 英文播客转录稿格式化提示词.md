# 英文播客转录稿格式化提示词

## 任务描述
将原始英文播客/视频转录稿（无格式文本）按照标准 Markdown 格式进行优化，使其结构清晰、易于阅读。

## 输入
- 原始转录稿文件（通常是长段无格式文本）
- 参考格式文件（可选，用于确定具体的格式风格）

## 输出
- 格式化后的 Markdown 文件
- 根据内容重新命名文件

---

## 格式化规则

### 1. 标题结构
```markdown
# 主标题（文章主题）

> 副标题/引言引用

---

## 一级章节标题

### 二级子标题（章节内的细分点）
```

### 2. 分隔线
- 在主要章节之间使用 `---` 分隔线
- 引言后和总结前使用分隔线

### 3. 强调格式
- **关键概念** 和 **重要结论** 使用加粗 `**text**`
- 引用语句使用 `> quote`
- 金句/总结句使用加粗

### 4. 列表格式
- 无序列表使用 `-` 符号
- 有序列表使用数字 `1. 2. 3.`
- 列表项之间保持适当间距

### 5. 段落处理
- 每段之间保留一个空行
- 长段落根据语义拆分为多段
- 去除原文中的换行符（转录稿中的行尾换行）

### 6. 内容结构建议
根据内容类型，建议包含以下章节：

**对于教育/知识类内容：**
- 引言（问题引入）
- 核心概念解释
- 具体方法/技巧（分点列出）
- 实践建议
- 总结
- 结束语

**对于演讲/访谈类内容：**
- 演讲者介绍
- 开场故事/引言
- 主要观点（分章节）
- 具体案例
- 总结与行动建议

---

## 处理步骤

1. **阅读原始文件**
   - 理解内容主题和核心观点
   - 识别内容结构（引言、主体、结论）

2. **确定文件名**
   - 根据内容主题重新命名
   - 格式：`主题标题.md`
   - 示例：`How Memory Works and How to Improve It.md`

3. **构建文档结构**
   - 添加主标题（#）
   - 添加副标题/引言引用（>）
   - 划分主要章节（##）
   - 细分小节（###）

4. **格式化内容**
   - 去除多余的换行符
   - 添加段落间距
   - 标记重点内容（加粗）
   - 转换列表格式

5. **添加分隔线**
   - 在主要章节之间添加 `---`

6. **检查与优化**
   - 确保原文内容完整保留
   - 检查格式一致性
   - 验证链接和引用

---

## 示例对比

### 原始格式（转录稿）
```
Hello my friend, today I
want to start with something

that many people feel but do
not know how to explain.
```

### 优化后格式
```markdown
# 文章标题

> 副标题描述

---

Hello my friend, today I want to start with something that many people feel but do not know how to explain.
```

---

## 注意事项

1. **保留原文** - 只调整格式，不修改原文内容
2. **语义分段** - 根据意思分段，而非机械分行
3. **层次分明** - 使用不同级别的标题体现内容层次
4. **重点突出** - 关键概念和金句使用加粗
5. **阅读友好** - 确保格式化后的文档易于阅读和导航

---

## 文件命名建议

- 使用英文标题（如果原文是英文）
- 首字母大写（Title Case）
- 去除无意义的哈希值文件名
- 示例：
  - ❌ `e9348ed826d64cc651fa834360ad2b82.md`
  - ✅ `How Memory Works and How to Improve It.md`
